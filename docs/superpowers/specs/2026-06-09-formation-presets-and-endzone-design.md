# 阵型预设 + 得分区视觉 设计

日期：2026-06-09
状态：已确认

## 背景与目标

两个相关需求：
1. **阵型预设**：在战术板内提供一组标准极限飞盘阵型，一键把全场 14 名球员 + 飞盘摆到对应站位，省去手动逐个拖。参考某手机战术板小程序的 5 种预设。
2. **得分区视觉**：当前左右得分区是明显的暗色块，改成更克制的样式——浅白线划分 + 端区内竖排浅白 "END ZONE" 文字，去掉暗色填充。

球场保持现有**横向**布局（得分区在左右）。参考图是竖向手机界面，阵型已按 90° 旋转映射到横向场上，由用户在可视化拖拽编辑器中亲手摆定，坐标见下。

## 一、得分区视觉（改 `client/src/components/Field.jsx`）

- **去掉**左右两块 `END_ZONE_COLOR`（`#1e3d1a`）暗色填充，整场统一 `FIELD_COLOR`（`#2d5a27`）
- 外框、左右端区线颜色从纯白 `#ffffff` 改为浅白 `rgba(255,255,255,0.45)`
- 左右端区内各加一个**竖排** "END ZONE" 文字：浅白 `rgba(255,255,255,0.25)`，Konva `Text` 旋转 90°，垂直居中放在各自端区中央（左端区中心 x≈0.09，右端区中心 x≈0.91）
- 中线虚线保留不变
- `Field` 仍是 `listening={false}` 的纯展示层

约定：红队（左）进攻、蓝队（右）防守、飞盘给红队持盘人。

### 2.2 入口（新组件 `FormationMenu`，挂在 `BoardCanvas` 底栏）

- 底栏一个「阵型 ∧」按钮，点击**向上弹出**菜单，列出 5 个预设的 label
- 选中一个即应用并关闭菜单；点菜单外区域关闭
- **播放中禁用**（与其它编辑入口一致，避免播放时改帧）

### 2.3 应用语义（新 store action `applyFormation(formationKey)`）

选一个预设时：
1. 作用于**当前可编辑帧**（沿用现有可编辑帧判定；播放中入口已禁用）
2. 对每个球员 `r1..r7` / `b1..b7`，把该帧 `playerStates[id]` 的 `x`、`y` 设为预设坐标；**保留 `orientation`**（朝向不动）；**删除 `ctrl`**（位置已重置，旧曲线作废）
3. 飞盘：把该帧 `discStates` 中**第一个飞盘**（`board.data.discs[0].id`）的 `x`、`y` 设为预设 `disc` 坐标，删除其 `ctrl`；**其余飞盘不动**（多盘场景）。若该帧无任何飞盘，跳过飞盘设置
4. 整个操作记**一步撤销历史**（`withHistory`），可一次 Undo 还原
5. 只改当前帧，不触碰其它关键帧
6. 球员/球队数量按现有 `board.data.players` 实际存在的 id 来设（若某 id 不存在则跳过，保持健壮）

## 技术实现概览

- **新增** `client/src/utils/formations.js`：`FORMATIONS`、`FORMATION_ORDER`（纯数据 + 可加一个纯函数 `buildFormationPatch(formationKey, players, discs)` 返回 `{playerStates, discStates}` 补丁，便于单测）
- **新增** `client/src/components/FormationMenu.jsx`：底栏弹出菜单（受控 open 状态、外点关闭、播放禁用）
- **修改** `client/src/store/boardStore.js`：加 `applyFormation(formationKey)`，用 `withHistory` 包裹，按 2.3 语义批量改当前帧的 `playerStates` / 第一个 `discStates`，清 `ctrl`
- **修改** `client/src/components/BoardCanvas.jsx`：底栏挂 `FormationMenu`，接 `applyFormation`，播放中禁用
- **修改** `client/src/components/Field.jsx`：去暗块、浅白线、竖排 END ZONE 文字

## 测试影响

- 新增 `client/src/utils/formations.test.js`：验证 `buildFormationPatch` 对各预设产出正确坐标、保留 orientation、删除 ctrl、只设第一个飞盘、跳过不存在的 id
- 新增 store 测试：`applyFormation` 设当前帧位置、记一步历史（Undo 可还原）、不动朝向、清曲线、不影响其它帧
- `Field.jsx` 视觉改动靠浏览器人工验收；如有快照/渲染测试需同步
- 现有动画/插值/标注测试不应受影响（只改帧内位置数据，结构不变）

## 非目标（本次不做）

- 不改球场朝向（保持横向）
- 不做"分进攻/防守两组分别套用"（整场快照）
- 不设球员朝向、不加阵型缩略图预览
- 不动多盘中第一个以外的飞盘
- 预设清单固定 5 个，不做自定义保存
