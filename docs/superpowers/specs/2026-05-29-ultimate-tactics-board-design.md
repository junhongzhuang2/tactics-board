# 极限飞盘战术板 — 设计文档

**日期：** 2026-05-29  
**状态：** 已确认  
**技术方案：** 方案 B（全栈 + Socket.io 实时协作）

---

## 1. 项目目标

为竞技极限飞盘（大学生联赛 / 俱乐部级别）提供一个基于网页的战术板工具，支持：
- 制作关键帧动画，讲解进攻/防守战术
- 手动复现比赛跑位，逐帧分析
- 通过链接分享给队员查看或协作编辑

---

## 2. 技术架构

### 前端
- **React**：UI 框架
- **Konva.js**：Canvas 2D 渲染，处理拖拽、图层、动画插值

### 后端
- **Node.js + Express**：REST API（创建/读取/保存战术板）
- **Socket.io**：实时协作同步
- **PostgreSQL**：持久化存储战术板数据

### 部署
- 开发阶段：本机 localhost 运行，无需服务器
- 上线阶段：国内轻量云服务器（腾讯云/阿里云，2核2G 实例）

---

## 3. 核心数据模型

```
Board（战术板）
  ├── id: 唯一标识（用于生成分享链接）
  ├── name: 战术板名称（如"vs XX队 进攻战术"）
  ├── editToken: 编辑权限 token
  ├── readToken: 只读权限 token
  ├── players: 球员配置（名字、队伍、编号）
  └── frames: Frame[]

Frame（关键帧）
  ├── index: 帧序号
  ├── duration: 过渡到下一帧的时长（毫秒）
  ├── playerStates: { playerId → { x, y, orientation } }
  ├── discState: { x, y }
  └── annotations: Annotation[]（单帧标注）

GlobalAnnotations（全局标注，不属于任何帧）
  └── Annotation[]

Annotation（标注）
  ├── type: 'dashed-arrow' | 'solid-arrow' | 'shape' | 'text'
  ├── scope: 'frame' | 'global'
  └── data: 坐标、颜色、文字等
```

所有球员/飞盘坐标使用**归一化坐标**（0-1，相对于场地宽高），渲染时乘以实际画布尺寸，保证跨设备一致。

## 4. 战术板画布

### 场地
- 标准极限飞盘场地（长约100m，宽约37m，两端各有端区）
- 画布自适应窗口，场地保持正确长宽比
- 缩放/平移为纯视图操作，不影响数据坐标

### 球员元素
- 红队（进攻/主队）：7人，默认编号 1-7
- 蓝队（防守/客队）：7人，默认编号 1-7
- 每名球员可单独重命名（替换编号显示）
- 圆形图标，队伍颜色区分，圆内显示编号或名字缩写
- 可选显示**视野锥**（半透明扇形）表示球员朝向
  - 全局开关控制是否显示
  - 选中球员后，通过旋钮或双指旋转调整朝向
  - 朝向数据纳入关键帧系统，每帧独立记录

### 飞盘元素
- 独立图标，可拖拽
- 与球员等同地位——独立记录坐标，参与关键帧和动画

### 标注工具（可收起工具栏）
- **虚线箭头**：传盘路线
- **实线箭头（可手绘曲线）**：球员跑动路线
- **圆圈 / 方框**：标注防守区域
- **文字**：说明文字
- 每条标注独立设置 scope：
  - `frame`：仅在当前帧显示
  - `global`：所有帧均显示

## 5. 关键帧动画系统

### 核心概念
每一帧是全场所有元素的**完整状态快照**（所有球员位置+朝向、飞盘位置、单帧标注）。

播放时，从当前帧到下一帧，所有改变了位置的元素**同步插值移动**（线性插值）。没有改变位置的元素停在原地不动。用户通过在目标帧拖动哪些元素，决定哪些人参与这一段动画。

### 时间轴（底部可视化进度条）
- 横向排列所有帧，每帧显示帧号
- 点击任意帧跳转；拖动进度指示器可 scrub
- 每帧可单独设置过渡时长（控制该帧动画速度）
- 控制按钮：播放 / 暂停 / 单步前进 / 单步后退

### 帧编辑
- 选中某帧 → 拖动球员/飞盘 → 自动写入该帧（无需手动保存）
- 可复制某帧作为下一帧的初始状态
- 可插入 / 删除 / 重排帧

## 6. 协作与分享

### 战术板管理
- 首页：展示所有战术板列表，支持命名、复制、删除

### 链接权限
- **编辑链接**（含 editToken）：可拖动、修改、添加标注
- **只读链接**（含 readToken）：仅能播放动画

### 实时协作
- 多人持有同一编辑链接时，通过 Socket.io 实时同步操作
- 无需注册登录，持有链接即可进入

### 性能约束
- 拖动时对 Socket 消息做 **Throttle（每 30-50ms 发送一次）**
- 拖动结束（mouseup / touchend）立即发送最终坐标并写入数据库
- 每条消息只发 diff（元素 ID + 新坐标），不发全量状态
